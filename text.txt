1 версия нашего файла
git init -используется для создания репозитория который следит за изьенениями данного файла или каталога
git remote add origin [url] - указываем url удаленного репозитория
git clone [url] - скачиваем удаленный репозитори
git status - показывает какие файлы мы сохранили в репозитории а какие нет
git add - можем сохронить отдельно выбранные файлы или можем все
2 версия нашего файла
git log - показывает наши комиты
git commit -создает комит
git branche -d [-D-принудителное удалени]- удаление локальной ветки
git push origin --delete [название ветки] удаление удаленной ветки
git stash - использкется для временного сохронения изменения
git marge - слияние веток с одним комит
git rebase - слияние веток с историей комитов
git reset HEAD~1- удаляет последний комит оставляет изменения
git reset --soft HEAD~1 - удаляет комит но оставляет его в индексе 
git remote remove project_name - удаляет удаленный репозиторий
rm -rf .git - удаляет скрытую папку git с методанныйми
git -rf [путь к директории] - удаляет локальную директорию с вашим гит репозиторием
git branche -vv и switch-branch - выводят дополнительную инфрормацию о ветках
git flow init - создается GitFlow в существующем репозитории
git flow feature start featureName - создается ветка от develope для разработки новой функции
git flow feature finish featureName - происходит слияние с develop и удаление featureName
git flow release start releaseName - ветка от develope перед добавлением в master (для тестирования и исправления)
git flow release finish releaseName - слияние с master и удаление ветки releaseName
git flow hotfix start hotfixName - ветка от master для исправления ошибок 
git flow hotfix finish  hotfixName - слияние с master и develope после исправление,также автоматически удаляется
